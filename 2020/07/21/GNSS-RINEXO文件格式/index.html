RINEXO格式说明

RINEX格式简介

RINEX（Receiver Independent Exchange Format/与接收机无关的交换格式）是一种在GPS测量应用中普遍采用的标准数据格式。

该格式采用文本文件存储数据，数据记录格式与接收机的制造厂商和具体型号无关

RINEX格式由瑞士伯尔尼大学天文学院（Astronomical Institute, University of Berne）的Werner Gurtner于1989年提出。当时提出该数据格式的目的是为了能够综合处理在EUREF89(欧洲一项大规模的GPS联测项目)中所采集的GPS数据。该项目采用了来自4个不同厂商共60多台GPS接收机。

RINEXO文件头说明

关键词 含义
RINEX VERSION / TYPE RINEX格式版本号
PGM / RUN BY / DATE 文件生成商信息
COMMENT 注释
MARKER NAME 测站名，测站标识符
MARKER NUMBER 测站标号
MARKER TYPE 标识类型
OBSERVER / AGENCY 观测机构的名称
REC # / TYPE / VERS 接收机的信息
ANT # / TYPE 接收机的天线类型
APPROX POSITION XYZ 测站粗略坐标
ANTENNA: DELTA H/E/N 天线ENH信息
SYS / # / OBS TYPES 各系统观测值类
INTERVAL 观测值间隔
TIME OF FIRST OBS 初始历元
TIME OF LAST OBS 结束历元
SYS / PHASE SHIFTS 系统相位偏差
END OF HEADER 结束标识

OBS TYPES详解

在多频多系统GNSS系统中，常用系统的频率分布如下图所示（图片来自Springer hander book）

这些多频信号，在被接收机接收解码后，每个频率对应的观测值就会按照一定的格式写入rinexo文件中，例如

E 16 C1C C5Q C7Q C8Q D1C D5Q D7Q D8Q L1C L5Q L7Q L8Q S1C SYS / # / OBS TYPES

  1. E : 观测值系统，Galileo
  2. 16：观测到的卫星数，16颗
  3. C1C：E1频率的C码

每个系统的观测值类似都如上述规律记录，C1C中，第一个C表示观测值类型，如C伪距码，D多普勒，L载波相位等。第二个1表示频点，Galileo常用频点为band1，band5，band7，band8。最后一位C表示观测值的解码类型。
